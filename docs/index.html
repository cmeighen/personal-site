smash.gg Weekly Tournament Digest Emailer

Dynamic Event Selection and Template Generation

This was my first project at smash.gg. The objective was to build a weekly emailer that would determine the best events to advertise to each user emailed, reaching ~225k users. The feature would select events for each user based on events they had previously attended, their location, and their user preferences. It would also filter out events they were already registered for.

Following the provided specs, I created mustache templates and a processing function to recursively build the user's email from smaller, component style templates rather than a single template.

In addition to the emailer itself, I created an admin tool to allow the partner support team to add, edit, and remove events from the week's eligible list, as well as specify target audiences for each event, circumventing the regular filters. Events could also be designated as "hero" events, which used a separate template to highlight it prominently within the email.

  • Mustache
  • React
  • MySQL

smash.gg Event Registration Emailer

Purchase Receipt, Registration Information, Google Map, and QR Codes

For this feature, I created functions to pull various user data, including purchase details, user registration information, event information, and payment records.

I also created a series of component-style Mustache templates using tables, as email styling using newer css options is not supported by most email clients.

With both of these in place, the data is processed into the templates and the email is assembled!

For this feature, I also created a QR code feature, allowing TOs to optionally generate unique QR codes for each registrant, that could later forward to their own specified link for event check-in purposes.
